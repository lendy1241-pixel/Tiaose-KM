# 调色训练版 v3.0 — KM引擎 使用说明

## 你现在拿到的

```
Tiaose-KM/
├── 调色-KM-v3.0.ipa      ← 已注入好 KMEngine 引用，就差 dylib
├── 使用说明.md            ← 你正在看的文件
│
├── KMHooks.m              ← 方法交换 (替换旧的 mixColor / checkTapped)
├── KMEngine.h/m           ← KM 混合引擎核心
├── KMPigment.h/m          ← 色浆光谱数据模型
├── KMPigmentDatabase.h/m  ← 8种预设色浆库
│
├── patch_macho.py         ← Mach-O dylib注入工具 (已用过, 备份用)
├── make_ipa.py            ← IPA 重新打包工具
├── build_dylib.sh         ← macOS 编译脚本
│
├── km_engine.py           ← Python 验证版 (可直接跑)
├── formula_solver.py      ← 配方求解器 (给定目标色 → 最优配方)
└── .github/workflows/     ← GitHub Actions 云端编译
```

## 方案 A: GitHub Actions 云端编译 (推荐, 免费)

### 1. 上传到 GitHub
将整个 `Tiaose-KM` 目录 push 到一个 GitHub 仓库 (可以是私有的)。

### 2. 自动编译
Push 后 GitHub Actions 自动触发编译。等 2 分钟。

### 3. 下载 dylib
进入 Actions → 点击最新的 workflow run → 下载 `KMEngine.dylib` artifact。

### 4. 打包 IPA
```bash
python3 make_ipa.py \
    --ipa build/调色-KM-v3.0.ipa \
    --dylib KMEngine.dylib \
    --out 调色-v3.0-完整版.ipa
```

### 5. 安装
用 TrollStore 安装 `调色-v3.0-完整版.ipa`。

## 方案 B: 有 Mac 直接编译 (最快)

```bash

./build_dylib.sh

# 2. 打包 IPA (dylib 会生成在 build/ 目录)

```

## 重要: 色浆名称必须匹配!

dylib 注入后，你在 App 里添加的色浆名称必须是以下之一才能被 KM 引擎识别:

| 色浆名称 | 描述 |
|----------|------|
| 钛白 | Titanium White — 必须加! 不加会很暗 |
| 炭黑 | Carbon Black |
| 铁红 | Red Iron Oxide (PR101) |
| 酞青蓝 | Phthalo Blue (PB15:3) |
| 铬黄 | Chrome Yellow (PY34) |
| 酞青绿 | Phthalo Green (PG7) |
| 永固红 | Permanent Red (PR170) |
| 群青 | Ultramarine Blue (PB29) |

如果色浆名称不在库中，该色浆会被静默忽略。
建议: 每个配方里始终加 200-500g 钛白作为基料。

## 预期效果

安装后，你的 App 会:
- `mixColor` → 用 KM 光谱混合 (不再是 RGB 加法)
- `checkTapped:` → 显示 CIEDE2000 色差 (ΔE00)
- 色差精度从 "玩具级" 升级到 "工业标准级"
- 不加钛白会得到很暗的颜色 (这是对的! 减法混色)
